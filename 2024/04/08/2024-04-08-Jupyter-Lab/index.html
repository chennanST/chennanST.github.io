使用Jupytter Lab进行远程开发

陈楠 Lv1
  2024-04-08 16:54:55 2024-04-08 16:54:55 Created   2024-04-14 11:08:03 2024-04-14 11:08:03 Updated  
  • linux 
 

Abstract

欢迎来到本教程，这篇博客将向您展示如何设置并访问远程服务器上的 JupyterLab。在远程开发中，使用 JupyterLab 可以极大地提高工作效率，并允许您在远程服务器上运行代码、处理数据和进行实验，而无需在本地安装大量软件包和依赖项。通过本教程，您将学习如何设置远程服务器上的 JupyterLab，并通过本地浏览器轻松访问它。

INTRODUCTION

随着远程工作和协作的普及，远程开发环境变得越来越重要。JupyterLab 是一个功能强大的交互式开发环境，可让您在浏览器中编写和运行代码、查看数据可视化，并以笔记本的形式记录您的工作流程。通过在远程服务器上设置 JupyterLab，您可以充分利用服务器的计算资源，而无需将数据或代码传输到本地机器。

在本教程中，我们将介绍如何通过 SSH 连接到远程服务器，设置 JupyterLab 并启动它，然后使用本地浏览器访问远程服务器上运行的 JupyterLab 实例。我们还将讨论一些安全性考虑，以确保您的远程开发环境是安全的，并且只有授权用户可以访问。

让我们开始设置您的远程 JupyterLab 环境吧！

3. 访问 Jupyter Lab：

在第1部分’启动远程 Jupyter Lab’之后，命令行窗口会显示一个地址。

命令行中的输出可能类似于

http://localhost:8888/?token=your_token_here

此时，如果已操作完第2部分’创建 SSH 隧道’，可按住Ctrl键点击此地址，直接跳转到本地浏览器访问远程Jupyter Lab。

一般情况下，localhost会是你的远程服务器的 IP 地址，如果不是，可以在浏览器弹出时，在地址栏修改它为your_server_ip

或者，可以打开本地浏览器，并访问 http://your_server_ip:8888/lab

此时会需要你输入token(令牌)，令牌就是命令行输出中的

your_token_here

看起来像一大段乱码

如果 Jupyter Lab 设置了密码，浏览器将提示你输入密码。

4. 使用 Jupyter Lab：

现在，你应该可以在本地浏览器中看到远程服务器上运行的 Jupyter Lab。你可以创建、编辑和运行代码，并与远程服务器进行交互。

注意事项：

  • 确保你的远程服务器上已经正确安装了 Jupyter Lab。
  • 确保远程服务器上的防火墙允许使用指定的端口（默认为 8888）。
  • 替换 username 和 your_server_ip 为你实际使用的用户名和远程服务器的 IP 地址。
  • 为了方便，你可以将 SSH 命令添加到本地机器上的脚本或别名中，以便更轻松地连接到远程 Jupyter Lab。
